## 💡 RESPOSTAS PRONTAS

| Pergunta | Resposta |
|----------|----------|
| **"Por que essa arquitetura?"** | Separação de responsabilidades. UI não conhece DB. Fácil testar e manter. |
| **"Como valida CPF?"** | Algoritmo real: módulo 11 com dois dígitos verificadores. Testado. |
| **"E se trocar banco?"** | Repository Pattern abstrai isso. Mudo MySQL para PostgreSQL sem alterar serviços. |
| **"Como testa sem DB?"** | Repositórios em-memória. Mesma interface, implementação diferente. |
| **"Qual a importância de interfaces?"** | Contrato bem definido. Múltiplas implementações. Desacoplamento. Testabilidade. |
| **"Por que exceções personalizadas?"** | Cada tipo tem contexto específico. Tratamento apropriado por tipo. |
| **"Como funciona o logging?"** | Centralizado: console (colorido) + arquivo (Logs/). Integrado em Program.cs |
